# AGENTS.md

这是《李宁品牌 AI 视觉规范项目》的项目规则。

## 项目定位

本项目用于沉淀李宁品牌在 AI 视觉生成中的品牌调性、品类规范、活动规范、节日规范、文案规则和视觉禁区。

本项目是其他李宁视觉项目的品牌规范依据。处理海报、社群图、电商图、素材图、3D、视频分镜或动态视觉任务时，应优先参考本项目。

## 必须参考的规范

处理李宁品牌 AI 视觉规范任务时，必须先根据用户需求判断品类、活动、节日等视觉触发条件，再按 `05_Codex-Workflow/Visual-Guideline-Routing-Rules.md` 调用对应规范。

- `01_Brand-Foundation`：品牌基础规范
- `02_Category-Visual-Guidelines`：品类与系列视觉规范
- `03_Campaign-Visual-Guidelines`：活动与大促视觉规范
- `04_Festival-Visual-Guidelines`：节日视觉规范
- `05_Codex-Workflow`：用户使用说明、常用指令和 Skill 说明文档

## 视觉规范加权融合规则

- 品牌基础规范不是最后兜底，而是每次视觉任务都必须参与。
- 品类 / 活动 / 节日规范决定专项视觉方向。
- 品牌基础规范负责品牌一致性、专业运动调性、字体底线、文案语气和禁区控制。
- 每次输出视觉方向或提示词前，必须说明本次调用了哪些规范，以及每个规范的大致权重。
- 不允许只调用专项规范而忽略品牌基础规范。
- 不允许只调用品牌基础规范而忽略需求中明确出现的品类、活动或节日规范。

权重原则：

- 单一品类任务：品类规范 60% + 品牌基础规范 40%。
- 单一活动任务：活动规范 60% + 品牌基础规范 40%。
- 单一节日任务：节日规范 60% + 品牌基础规范 40%。
- 活动 + 品类任务：活动规范 50% + 品类规范 30% + 品牌基础规范 20%。
- 节日 + 品类任务：节日规范 50% + 品类规范 30% + 品牌基础规范 20%。
- 节日 + 活动任务：主主题规范 50% + 次主题规范 30% + 品牌基础规范 20%；主主题不明确时，节日规范 50% + 活动规范 30% + 品牌基础规范 20%。
- 节日 + 活动 + 品类任务：主主题规范 45% + 次主题规范 25% + 品类规范 15% + 品牌基础规范 15%；主主题不明确时，节日规范 45% + 活动规范 25% + 品类规范 15% + 品牌基础规范 15%。
- 仅品牌任务：品牌基础规范 100%。

## 需求识别规则

- 不允许自行猜测品类、活动或节日。
- 不允许自动补全需求中没有的信息。
- 如果需求未提供某个维度，必须标记为“需求未提供”。
- 每次输出视觉方向前，必须列出【本次需求识别】【本次规范调用权重】【融合后的视觉方向】。

## Skill 文件关系

- `.agents/skills/li-ning-visual-director/SKILL.md` 是 Codex 正式调用的机器版 Skill。
- `05_Codex-Workflow/Li-Ning-Visual-Director-Skill.md` 是给用户阅读的人类说明版 Skill。
- 两份 Skill 规则应保持一致；如果规则更新，应同步维护。

## 核心禁区

- 不要编造官方品牌手册、官方色值、官方字体、官方口号。
- 不确定的官方信息必须标注“待官方资料确认”。
- 输出海报、社群图、电商图、素材图、3D、视频分镜或动态视觉提示词时，必须明确品牌调性、字体判断、文案是否入画、品牌禁区和检查表。
- 如果用户提供主标题、slogan、权益、价格、产品名等文案，必须判断是否需要入画，并提醒中文、数字、价格、货号存在 AI 生成错误风险。
- 产品、Logo、字体和官方视觉资产如无法确认来源，不得伪装成官方标准。

## 被其他项目调用时

如果本项目被 `Li-Ning-Multi-Grid-Visual-Guidelines` 或其他李宁项目调用，本项目作为品牌调性与视觉规范依据。

多宫格结构、参考图模板、素材库索引和项目执行规则，应优先遵守调用方项目内的 `AGENTS.md`。
