# 李宁品牌 AI 视觉规范调用优先级规则

## 总原则

Codex 处理李宁视觉任务时，必须先读取用户需求，判断需求中明确出现了哪些视觉触发条件：

- 品类：跑步、篮球、儿童、训练、户外、运动生活等。
- 活动：618、双11、奥莱、新品首发、会员日、直播活动等。
- 节日：端午、中秋、春节、国庆、七夕、儿童节、父亲节、母亲节等。
- 品牌基础：李宁品牌 DNA、品牌调性、禁区、字体方向、文案风格等。

只能根据用户明确提供的需求内容判断，不允许自行猜测品类、活动或节日。

如果需求中没有明确写出某项内容，必须标记为：

```text
需求未提供
```

不要自动补全需求中没有的内容。

## 品牌基础规范的定位

`01_Brand-Foundation` 不是“最后兜底才看”的规则，而是所有视觉任务的共同基础层。

品牌基础规范必须与品类规范、活动规范、节日规范进行加权融合，共同决定最终视觉方向。

品牌基础规范负责：

- 李宁整体品牌 DNA。
- 专业运动品牌调性。
- 中国运动品牌识别。
- 年轻化、速度感、能量感、功能科技感。
- 字体底线与文案语气。
- 品牌禁区。
- 不确定官方信息的处理方式。

不允许只调用专项规范而忽略品牌基础规范。
也不允许只调用品牌基础规范而忽略需求中明确出现的品类、活动或节日规范。

## 视觉规范加权融合规则

### 1. 单一品类任务

如果需求明确是品类任务，例如跑步、篮球、儿童、训练、户外、运动生活：

```text
对应品类规范：60%
品牌基础规范：40%
```

示例：

```text
跑步品类海报
→ Running-Category 60%
→ Brand Foundation 40%
```

说明：品类规范决定速度感、场景、品类视觉语言、产品表现方式；品牌基础规范决定李宁整体品牌气质、字体底线、文案语气、品牌禁区和专业运动品牌调性。

### 2. 单一活动任务

如果需求明确是活动任务，例如 618、奥莱、双11、新品首发、会员日、直播活动：

```text
对应活动规范：60%
品牌基础规范：40%
```

示例：

```text
618 活动海报
→ 618-Campaign 60%
→ Brand Foundation 40%
```

说明：活动规范决定促销氛围、活动节奏、信息强度和大促视觉冲击力；品牌基础规范决定画面不能廉价，不能偏离李宁专业运动品牌气质。

### 3. 单一节日任务

如果需求明确是节日任务，例如端午、中秋、春节、国庆、七夕、儿童节、父亲节、母亲节：

```text
对应节日规范：60%
品牌基础规范：40%
```

示例：

```text
端午节海报
→ Dragon-Boat-Festival 60%
→ Brand Foundation 40%
```

说明：节日规范决定节日元素、节日氛围、文化符号和情绪表达；品牌基础规范决定李宁品牌调性、运动感、专业感和品牌限制。

### 4. 活动 + 品类任务

如果需求同时包含活动和品类，例如“618 跑步海报”“奥莱篮球海报”：

```text
活动规范：50%
品类规范：30%
品牌基础规范：20%
```

示例：

```text
618 跑步海报
→ 618-Campaign 50%
→ Running-Category 30%
→ Brand Foundation 20%
```

说明：活动规范决定促销主氛围；品类规范决定运动品类表达；品牌基础规范控制整体品牌一致性和禁区。

### 5. 节日 + 品类任务

如果需求同时包含节日和品类，例如“端午跑步海报”“父亲节儿童品类海报”：

```text
节日规范：50%
品类规范：30%
品牌基础规范：20%
```

示例：

```text
端午跑步海报
→ Dragon-Boat-Festival 50%
→ Running-Category 30%
→ Brand Foundation 20%
```

说明：节日规范决定主题情绪和节日符号；品类规范决定产品和运动场景表达；品牌基础规范保持李宁品牌一致性。

### 6. 节日 + 活动任务

如果需求同时包含节日和活动，例如“端午奥莱海报”“春节会员日海报”：

如果用户明确主主题：

```text
主主题规范：50%
次主题规范：30%
品牌基础规范：20%
```

如果用户没有明确主主题，则默认：

```text
节日规范：50%
活动规范：30%
品牌基础规范：20%
```

示例：

```text
端午奥莱海报，主主题是端午
→ Dragon-Boat-Festival 50%
→ Outlet-Special 30%
→ Brand Foundation 20%
```

```text
端午奥莱海报，主主题是奥莱
→ Outlet-Special 50%
→ Dragon-Boat-Festival 30%
→ Brand Foundation 20%
```

### 7. 节日 + 活动 + 品类任务

如果需求同时包含节日、活动和品类，例如“端午 618 跑步九宫格”：

如果用户明确主主题：

```text
主主题规范：45%
次主题规范：25%
品类规范：15%
品牌基础规范：15%
```

如果用户没有明确主主题，则默认：

```text
节日规范：45%
活动规范：25%
品类规范：15%
品牌基础规范：15%
```

示例：

```text
端午 618 跑步海报，主主题未明确
→ Dragon-Boat-Festival 45%
→ 618-Campaign 25%
→ Running-Category 15%
→ Brand Foundation 15%
```

### 8. 仅品牌任务

如果需求没有明确品类、活动、节日，只是李宁品牌形象类或普通品牌海报：

```text
品牌基础规范：100%
```

并标记：

```text
品类：需求未提供
活动：需求未提供
节日：需求未提供
```

## 输出要求

每次输出视觉方向或提示词前，必须先列出：

```text
【本次需求识别】
品类：
活动：
节日：
主主题：

【本次规范调用权重】
调用规范 1：xx%
调用规范 2：xx%
调用规范 3：xx%

【融合后的视觉方向】
视觉语言：
色彩方向：
字体方向：
场景方向：
文案语气：
禁区提醒：
```

如果需求未提供某个维度，必须标记为“需求未提供”，不要自动补全。

## 禁止事项

- 不允许 Codex 自行猜测品类、活动或节日。
- 不允许自动补全需求中没有的信息。
- 不允许只用品牌基础规范替代专项规范。
- 不允许只用专项规范而忽略品牌基础规范。
- 不允许把品牌基础规范描述为“最后兜底才用”。
