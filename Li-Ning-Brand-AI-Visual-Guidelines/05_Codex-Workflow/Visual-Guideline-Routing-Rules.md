# 李宁品牌 AI 视觉规范调用优先级规则

## 总原则

Codex 处理李宁视觉任务时，必须先读取用户需求，判断需求中明确出现了哪些视觉触发条件：

- 品类：跑步、篮球、儿童、训练、户外、运动生活等。
- 活动：618、双11、奥莱、新品首发、会员日、直播等。
- 节日：端午、中秋、春节、国庆、七夕、儿童节、父亲节、母亲节等。
- 品牌基础：李宁品牌 DNA、品牌调性、禁区、字体方向、文案风格等。

只能根据用户明确提供的需求内容判断，不允许自行猜测品类、活动或节日。

如果需求中没有明确写出某项内容，必须标记为：

```text
需求未提供
```

不要自动补全需求中没有的内容。

## 品类任务调用顺序

如果需求明确是品类任务，例如跑步海报、篮球海报、儿童产品海报、训练装备海报：

第一优先级：

```text
02_Category-Visual-Guidelines 中对应品类规范
```

第二优先级：

```text
01_Brand-Foundation 品牌基础规范
```

第三优先级：

```text
05_Codex-Workflow 中的输出规则与 Skill 说明
```

示例：

```text
需求写明“跑步品类”
→ 优先读取 Running-Category
→ 再读取 Brand Foundation
→ 不要只用泛泛的品牌基础风格替代跑步品类规则
```

## 活动任务调用顺序

如果需求明确是活动任务，例如 618、奥莱、双11、新品首发、会员日、直播活动：

第一优先级：

```text
03_Campaign-Visual-Guidelines 中对应活动规范
```

第二优先级：

```text
如果需求同时给出品类，则读取 02_Category-Visual-Guidelines 中对应品类规范
```

第三优先级：

```text
01_Brand-Foundation 品牌基础规范
```

第四优先级：

```text
05_Codex-Workflow 中的输出规则与 Skill 说明
```

示例：

```text
需求写明“618 跑步品类”
→ 优先读取 618-Campaign
→ 再读取 Running-Category
→ 最后读取 Brand Foundation
```

示例：

```text
需求写明“奥莱多产品促销”
→ 优先读取 Outlet-Special
→ 如果有品类，再读取对应品类规范
→ 最后读取 Brand Foundation
```

## 节日任务调用顺序

如果需求明确是节日任务，例如端午、中秋、春节、国庆、七夕、儿童节、父亲节、母亲节：

第一优先级：

```text
04_Festival-Visual-Guidelines 中对应节日规范
```

第二优先级：

```text
如果需求同时给出品类，则读取 02_Category-Visual-Guidelines 中对应品类规范
```

第三优先级：

```text
如果需求同时给出活动，则读取 03_Campaign-Visual-Guidelines 中对应活动规范
```

第四优先级：

```text
01_Brand-Foundation 品牌基础规范
```

第五优先级：

```text
05_Codex-Workflow 中的输出规则与 Skill 说明
```

示例：

```text
需求写明“端午跑步海报”
→ 优先读取 Dragon-Boat-Festival
→ 再读取 Running-Category
→ 最后读取 Brand Foundation
```

## 节日 + 活动 + 品类同时出现时

如果需求同时包含节日、活动和品类，例如：

```text
端午 618 跑步九宫格
```

调用顺序为：

第一优先级：

```text
用户需求中明确的主主题
```

如果主主题是端午，则节日优先。

如果主主题是 618，则活动优先。

如果需求没有明确主主题，则默认：

1. 节日规范
2. 活动规范
3. 品类规范
4. 品牌基础规范
5. Codex 工作流与 Skill 说明

输出中必须说明：

```text
本次视觉调用优先级依据：需求中同时出现节日 / 活动 / 品类，因此按节日 / 活动 / 品类 / 品牌基础进行组合调用。
```

## 没有明确品类、活动、节日时

如果需求没有明确品类、活动、节日：

第一优先级：

```text
01_Brand-Foundation 品牌基础规范
```

第二优先级：

```text
05_Codex-Workflow 输出规则
```

并标记：

```text
品类：需求未提供
活动：需求未提供
节日：需求未提供
```

## 品牌基础规范的定位

`01_Brand-Foundation` 不是每次都必须作为第一视觉层，而是所有任务的底层兜底规则。

品牌基础规范负责：

- 李宁品牌 DNA。
- 品牌调性。
- 品牌关键词。
- 品牌禁区。
- 文案语气。
- 基础字体方向。
- 不确定信息的处理方式。

无论优先读取哪个品类、活动、节日规范，最终都必须符合品牌基础规范。

## 输出要求

每次输出视觉方向或提示词前，必须列出：

- 识别到的需求类型。
- 调用的规范文件。
- 调用优先级。
- 未提供的信息。

如果需求未提供某个维度，必须标记为“需求未提供”，不要自动补全。
