# Reference Match Check

## 检查项
- 是否只读取了需求表或需求图片中明确出现的内容。
- 是否出现了需求中没有的文案。
- 是否出现了需求中没有的价格。
- 是否出现了需求中没有的权益。
- 是否出现了需求中没有的券包、折扣、促销标签。
- 是否出现了需求中没有的人物、场景、卖点。
- 是否把参考图中的文字误当成需求文案。
- 是否把李宁品牌规范中的通用词误当成运营文案。
- 如果需求缺失，是否标记为“需求未提供”。
- 如果缺失内容影响输出，是否提醒用户确认。
- 是否启用了参考图强锁定模式。
- 强锁定模式下，是否严格保持整体构图不变。
- 强锁定模式下，是否严格保持宫格结构不变。
- 强锁定模式下，是否严格保持模块位置不变。
- 强锁定模式下，是否严格保持产品位置不变。
- 强锁定模式下，是否严格保持二维码位置不变。
- 强锁定模式下，是否严格保持文字区域位置不变。
- 强锁定模式下，是否严格保持信息层级结构不变。
- 强锁定模式下，是否只修改了配色、字体设计、元素内容、风格调性、人物素材和背景。
- 是否错误调整了结构骨架。
- 是否错误调整了格子功能。
- 最终提示词是否仍然包含本地路径。
- 是否已经把参考图转写为版式结构描述。
- 最终提示词复制到外部 AI 平台后是否仍然可理解。
- 最终提示词是否可以直接复制到其他 AI 生图平台。
- 是否使用了用户指定参考图。
- 是否误用了其他参考图。
- 如果有多张候选参考图，是否先让用户确认。
- 最终结构描述是否包含主标题区、产品区、二维码区、每格功能和信息层级。
- 当用户要求参考图时，是否将参考图作为结构布局参考，而不是普通风格参考。
- 是否严格保留参考图的信息布局。
- 是否严格保留参考图模块关系。
- 是否严格还原参考图整体构图。
- 是否严格保持参考图模块位置。
- 是否严格保持参考图产品位置和货品陈列位置。
- 是否严格保持参考图二维码位置。
- 是否严格保持参考图文字区域位置。
- 是否严格保持参考图信息层级结构。
- 参考图中的价格条是否只作为结构参考。
- 如果没有价格文案，是否没有擅自生成价格类文字。
- 如果保留底部条形区域，是否为无文字结构或视觉分隔区。
- 是否没有照搬参考图原风格。
- 是否没有照搬参考图原配色。
- 是否没有照搬参考图原背景。
- 是否没有照搬参考图原人物。
- 是否没有照搬参考图原光影和装饰元素。
- 是否已经结合李宁品牌规范重新设计视觉风格。
- 是否只调整基于李宁品牌规范、活动主题、品类特点和用户需求重塑后的视觉风格、配色、品牌调性、场景氛围和具体产品内容。
- 是否错误改变版式骨架。
- 是否随意移动主标题区、产品区、二维码区、价格权益区或行动引导区。
- 是否说明相似模板编号。
- 是否需要新增模板编号。
- 是否拆解每格作用和整体结构。

## 一票否决

如果用户明确要求参考某张多宫格参考图，但最终提示词改变了参考图的结构骨架、模块位置、产品陈列位置、二维码位置或文字区域位置，本次检查即判定为不通过。

如果参考图处于强锁定模式，但整体构图、宫格结构、模块位置、产品位置、二维码位置、文字区域位置、每格功能、信息层级结构或拼接关系发生改动，本次检查即判定为不合格。

如果最终提示词中仍出现 `G:\Reference-Library` 这类本地路径，本次检查即判定为不通过。

如果最终提示词要求照搬原图风格、完全复刻参考图视觉风格，或明显复用原图配色、背景、人物、光影和装饰元素，本次检查即判定为不通过。

如果参考图中的价格区、黄色条、权益条或底部信息条被错误复用为文案内容，而非仅作为结构位置参考，本次检查即判定为不通过。

如果出现自动补全或自动推断，本次检查即判定为不合格。
