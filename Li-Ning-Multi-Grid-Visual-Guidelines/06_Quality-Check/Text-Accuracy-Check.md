# Text Accuracy Check

## 检查项
- 是否只读取了需求表或需求图片中明确出现的内容。
- 画面中所有中文文案是否全部来自运营需求表或用户明确提供内容。
- 是否存在 Codex 自行新增、改写、扩写的中文文案。
- 是否出现了需求中没有的文案。
- 是否出现了需求中没有的价格。
- 是否出现了需求中没有的权益。
- 是否出现了需求中没有的券包、折扣、促销标签。
- 是否出现了需求中没有的人物、场景、卖点。
- 是否把参考图中的文字误当成需求文案。
- 是否把李宁品牌规范中的通用词误当成运营文案。
- 如果需求缺失，是否标记为“需求未提供”。
- 如果缺失内容影响输出，是否提醒用户确认。
- 主标题、副标题、权益、价格、产品名、货号、行动引导是否逐字准确。
- 用户只提供“文案方向”时，是否被错误扩写成入画文案。
- 未提供的文案是否只做文字区预留，并标注“文案待运营确认”。
- 是否存在错字、漏字、多字、乱码。
- 价格、货号、权益、产品名是否与运营需求完全一致。
- 是否出现多余中文文案，例如“扫码查看更多”“立即购买”“装备精选”等未提供内容。
- 正式发布前是否建议 PS / Figma 人工复核并精修中文、数字、价格、货号和产品名。
- 是否所有文案均来自运营需求表或用户明确提供内容。
- 是否出现需求表没有提供的价格 / 权益 / 折扣 / 券包文案。
- 是否错误生成了“预估到手价”“到手价”“券后价”等文案。
- 如果没有价格文案，是否没有擅自生成价格类文字。
- 是否严格按照需求执行，没有自行补充营销话术。
- 强锁定模式下，是否错误新增了需求中没有的文案。
- 强锁定模式下，是否在没有价格文案时仍生成了价格类文字。

## 一票否决

只要出现未由运营需求表或用户明确提供的新增中文文案，本次提示词或画面检查即判定为不通过。

只要出现需求表没有提供的价格、权益、折扣、券包、促销标签、产品名、货号或价格数字，本次提示词或画面检查即判定为不通过。

只要出现自动补全或自动推断，本次提示词或画面检查即判定为不合格。
