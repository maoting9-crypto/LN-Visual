# Grid Rules

## 用途

本目录用于定义《李宁朋友圈多宫格视觉规范项目 V1.0》中九宫格、六宫格、四宫格、三宫格和双图组合的比例、编号、位置、发布顺序、裁切导出和二维码区域规则。

## 默认输出方式

- AI 出图提示词默认服务于一张完整多宫格海报。
- 格子编号用于规划完整海报内部结构，不代表默认要分别生成多张图片。
- 只有用户明确要求切图、单格导出或分别生成时，才输出每格独立提示词。

## 二维码区域宫格规则

1. 二维码不默认独占一个完整格子。
2. 二维码具体位置、大小、占比、所在格子或所在模块，必须以用户提供的参考图为准。
3. 如果参考图中二维码占据完整一格，则保留完整一格的二维码区域。
4. 如果参考图中二维码只占某个格子的一部分，则只保留该局部区域，不扩展成完整格子。
5. 如果参考图中二维码嵌入在某个信息模块中，则保持该嵌入关系。
6. 不允许因为需求中需要二维码，就重新调整宫格结构。
7. 不允许因为需求中需要二维码，就新增一个参考图中不存在的独立二维码格。
8. 不允许因为需求中需要二维码，就改变原有产品区、标题区、人物区、权益区的位置。
9. 如果需求明确需要二维码，画面中只在对应位置写“此处二维码区域”。
10. 不生成真实二维码图案。
11. 如果需求明确不需要二维码，则不生成“此处二维码区域”。
12. 如果需求没有明确是否需要二维码，则标记为“需求未提供”，不要自动添加二维码区域。

## 不同宫格的二维码处理

- 九宫格：二维码可以是独立一格，也可以是某一格内的局部模块；具体位置以参考图为准；不允许默认固定为第 6 格、第 8 格或第 9 格。
- 六宫格：二维码可以嵌入任意格子的局部信息区；不允许默认强制放在最后一格；具体位置以参考图为准。
- 四宫格：每格为 3:4 竖版结构时，二维码可以位于某一格内部角落或底部信息区；不允许强制占据完整一格；具体位置以参考图为准。
- 三宫格 / 双图组合：二维码可以作为局部信息模块出现；不允许为了二维码破坏原有图组结构；具体位置以参考图为准。

## 第一阶段范围

只做规划和提示词规则，不写自动切图脚本。
