# 6Grid Rules

## 比例

- 形式：2x3 或 3x2。
- 每格：根据参考图和发布平台保持一致比例。
- 可做连续背景，也可做产品推荐矩阵。
- 默认输出一张完整六宫格海报，内部按 2x3 或 3x2 规划，不默认分别生成 6 张图。

## 2x3 编号

1. 左上
2. 右上
3. 中左
4. 中右
5. 左下
6. 右下

## 3x2 编号

1. 左上
2. 上中
3. 右上
4. 左下
5. 下中
6. 右下

## 二维码区域规则

六宫格中的二维码不默认独占一个完整格子。

- 二维码可以嵌入任意格子的局部信息区。
- 不允许默认强制放在最后一格。
- 二维码具体位置、大小、占比、所在格子或所在模块，必须以用户提供的参考图为准。
- 如果参考图中二维码占据完整一格，则保留完整一格的二维码区域。
- 如果参考图中二维码只占某个格子的一部分，则只保留该局部区域，不扩展成完整格子。
- 如果参考图中二维码嵌入在某个信息模块中，则保持该嵌入关系。
- 不允许因为需求中需要二维码，就重新调整六宫格结构。
- 不允许因为需求中需要二维码，就新增一个参考图中不存在的独立二维码格。
- 不允许因为需求中需要二维码，就改变原有产品区、标题区、人物区、权益区的位置。
- 如果需求明确需要二维码，画面中只在对应位置写“此处二维码区域”。
- 不生成真实二维码图案。
- 如果需求明确不需要二维码，则不生成“此处二维码区域”。
- 如果需求没有明确是否需要二维码，则标记为“需求未提供”，不要自动添加二维码区域。
