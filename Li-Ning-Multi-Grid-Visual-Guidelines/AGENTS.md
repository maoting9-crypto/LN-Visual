# AGENTS.md

## 项目定位

本项目是《李宁朋友圈多宫格视觉规范项目 V1.0》，用于处理李宁朋友圈九宫格、六宫格、四宫格、三宫格、双图组合等多宫格视觉任务。

本项目只保存规则、索引、模板拆解、分析报告和提示词，不默认保存原始图片素材。

## 必须优先读取

处理任何多宫格任务前，必须先读取：

```text
AGENTS.md
.agents/skills/li-ning-multi-grid-director/SKILL.md
E:\LN-Visual\Li-Ning-Brand-AI-Visual-Guidelines
```

其中品牌规范项目 `E:\LN-Visual\Li-Ning-Brand-AI-Visual-Guidelines` 负责李宁品牌调性、字体判断、文案入画规则、品类视觉规范、活动视觉规范、节日视觉规范、品牌禁区、产品比例与文案准确性规则。

## 文案强约束

- 所有画面文案必须严格来自运营需求表或用户明确提供的文案。

- 如果用户只提供“文案方向”，必须标注为方向，不得直接扩写成入画文案。
- 如果画面需要文案但运营未提供，必须写明“文案待运营确认”，并在提示词中要求预留文字区域。

- 如果需求表没有提供价格、权益、折扣、券包等信息，最终提示词中不得出现任何价格类、优惠类、权益类文案。
- 如果需求表没有提供对应文案，不要写“预估到手价”“到手价”“券后价”等占位文字。
- 如果参考图中存在价格条、黄色条、权益条、促销标签区，只能作为“版式结构位置”参考，不能复用其文字内容。

## 参考图价格区处理规则

- 参考图中的价格区、黄色条、权益条、底部信息条，属于“信息模块位置参考”，不是“文案内容参考”。
- 如果运营需求表提供了价格 / 权益 / 折扣 / 券包文案，可以保留该信息区，但必须填入需求表原文，不得改写。
- 如果运营需求表没有提供价格 / 权益 / 折扣 / 券包文案，不得生成价格类文字，不得生成优惠类文字，不得生成“预估到手价”等占位文字。
- 如果运营需求表没有提供价格 / 权益 / 折扣 / 券包文案，可以根据用户是否要求结构强约束，保留无文字的信息承载区域 / 色块结构。
- 无文案的信息区只能描述为“无文字装饰条”“空白信息承载区”“视觉分隔色块”，不能写具体促销文案。
- 如果用户明确要求“完全不要价格区”，必须移除该模块，不得保留。
- 如果用户要求“严格保持参考图结构”，但没有提供价格文案，则保留价格区的位置和形状，但内部不放任何价格、权益、折扣文字。

- 当用户要求参考 `G:\Reference-Library\01_Multi-grid-reference` 中的参考图时，参考图必须作为“结构布局参考”，不是普通风格参考。
- 最终输出必须严格保持参考图的整体构图、模块位置、产品位置、货品陈列位置、二维码位置、文字区域位置和信息层级结构。
- 不允许照搬参考图原本的风格、配色、背景、人物、光影、品牌气质和装饰元素。
- 只允许结合李宁品牌规范、活动主题、品类特点和用户需求重新设计视觉风格、配色、品牌调性、场景氛围和具体产品内容。
- 不允许改变参考图的版式骨架，不允许随意移动主标题区、产品区、二维码区、价格权益区和行动引导区。

- 如果参考图结构与用户需求冲突，必须先标注冲突点并请求确认，不能擅自改版式。

## 参考图强锁定模式规则

- 如果用户明确要求“严格按照参考图来做”，参考图必须进入强锁定模式。
- 强锁定模式下，Codex 必须把参考图视为“版式母版”，不是普通参考图。
- 强锁定模式的本质是：结构不变，视觉重设计。

强锁定模式下，只允许修改：

- 配色。
- 字体设计。
- 图片内元素内容。
- 整体风格调性。
- 人物素材。
- 背景。

强锁定模式下，不允许改动：

- 整体构图。
- 宫格结构。
- 每格的位置。
- 各模块的位置。
- 产品摆放位置。
- 货品陈列逻辑。
- 主标题区位置。
- 副标题区位置。
- 权益 / 价格区位置；如果需求中没有文案，只保留位置，不生成文字。
- 二维码区位置。
- 文字区域位置。
- 每格功能关系。
- 信息层级结构。
- 背景延展关系。
- 拼接关系。

最终生成提示词时，必须明确写出：

- 严格保持参考图整体构图。
- 严格保持模块位置。
- 严格保持产品位置。
- 严格保持二维码位置。
- 严格保持文字区域位置。
- 严格保持每格功能和信息层级。
- 只重塑视觉风格，不重构版式结构。

- 不默认输出多张单独图片，不要求每个格子单独生成。
- 内部格子规划只用于说明这张完整海报里的版式结构，不等于要分别出多张图。

- 生成提示词时必须写清楚：最终输出为单张完整图片，画面内部按照指定宫格结构规划。
- 九宫格默认写为：一张完整 1:1 方形九宫格海报，内部为 3x3 网格布局，最终输出为单张完整图片。

## 二维码规则

- 如果多宫格海报需要二维码，二维码必须单独占用一个完整格子。
- 二维码不应随意贴在角落、压在产品上、压在人物上或跨越两个格子。
- 二维码格要预留干净背景、安全边距和明确扫码提示。
- AI 不建议直接生成可用二维码；提示词应写明“二维码位置预留，正式发布时后期替换真实二维码”。
- 二维码摆放需参考 `G:\Reference-Library\01_Multi-grid-reference` 中已入库参考图：常见位置为中心格、下中格、左中格或右中格，其中中心格适合强转化，底部中间格适合不打断主视觉。
- 如果二维码进入中心格，第 5 格优先作为二维码 / 核心权益 / 扫码引导区域。
- 如果中心格需要放主标题或核心产品，二维码可放第 8 格下中位置，保持扫码入口清晰。
- 如果参考图是非多宫格竖版榜单或单张社群图，二维码可能出现在角落；但只要任务是多宫格海报，仍必须按“二维码独占一个完整格子”执行。
