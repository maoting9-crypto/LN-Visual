# AGENTS.md

这是《李宁朋友圈多宫格视觉规范项目 V1.0》的项目规则。

## 项目定位

本项目是李宁朋友圈多宫格项目执行系统，用于把运营需求、参考素材和李宁品牌规范转化为完整多宫格海报提示词。

默认输出是一张完整多宫格海报，不默认拆成每格单独生图。

```text
E:\LN-Visual\Li-Ning-Brand-AI-Visual-Guidelines
```

## 多宫格参考素材学习规则｜修正版

参考素材用于学习布局规则，不是学习风格。

学习布局规则时，不能自由重构版式。如果用户明确想要某张参考图的排版布局效果，Codex 必须保留该参考图的布局关系。

这里的“学习并保留”必须理解为：参考素材就是本次海报的版式骨架。最终提示词必须要求外部 AI 平台严格保持参考素材的结构层，只替换视觉层。

结构层必须保持不变：

- 宫格数量、宫格比例、宫格排列方式。
- 每个格子的功能分工。
- 主标题、副标题、产品信息、二维码、产品卡的位置。
- 产品卡内部的人物、产品、标签、文字和背景的相对位置。
- 二维码所在格子、二维码在格子内的位置和占比。
- 背景跨格延展关系和裁切关系。
- 跨格连续人物 / 场景 / 产品大画面的拼接关系。

如果参考素材中多个格子共同组成一张连续人物场景、连续背景场景或连续产品穿着场景，最终提示词必须说明这些格子不是独立小图，而是同一张连续完整场景被宫格分隔线切开。不得把连续人物场景拆成每格独立人物图，不得把连续背景拆成每格独立背景。

视觉层允许替换：

- 配色。
- 字体设计。
- 产品内容。
- 人物素材。
- 背景素材。
- 节日元素。
- 品牌视觉风格。
- 场景氛围。

最终提示词中不要只写“参考”“借鉴”“学习大概布局”。必须写清楚“严格保持参考素材的宫格结构、模块位置、产品卡排列、二维码位置、信息层级和裁切关系，只替换配色、字体、产品、人物、背景、节日元素、品牌视觉风格和场景氛围”。

所谓“学习”，不是只提取大概规律，而是要准确学习并保持：

- 整体宫格结构。
- 每格功能关系。
- 主标题区位置。
- 副标题区位置。
- 产品区位置。
- 产品标签位置。
- 鞋品摆放位置。
- 模特 / 人物素材位置。
- 场景素材使用位置。
- 二维码区域位置。
- 信息层级结构。
- 背景延展关系。
- 裁切规则。

只允许根据李宁品牌需求重新调整：

- 配色。
- 字体设计。
- 产品内容。
- 人物素材。
- 背景素材。
- 节日元素。
- 品牌视觉风格。
- 场景氛围。

不允许自由改变：

- 版式骨架。
- 宫格结构。
- 每格功能。
- 产品卡排列方式。
- 二维码位置。
- 标题区位置。
- 产品标签位置。
- 模特和产品的相对关系。
- 信息层级结构。

不允许把六宫格参考图学习后改造成另一种六宫格结构。
不允许把产品卡结构重新设计成另一种产品展示方式。
不允许把参考素材的模块关系改成新的模块关系。
不允许把参考素材当作普通情绪板或风格参考。
不允许用“重新设计版式”“优化布局”“自由排版”“创意重组”等表达。

## 二维码区域规则

- 如果参考素材中二维码在第 1 格内部，最终也必须在第 1 格内部。
- 如果参考素材中二维码只占某一格的局部区域，最终也只能保留为局部区域。
- 不允许默认让二维码独占一个完整格子。
- 不允许把二维码移动到其他格子。
- 不允许把局部二维码区域放大成完整独立格，除非参考素材本身就是完整独立格。
- 需要二维码时，只在对应位置标注“此处二维码区域”。
- 不生成真实二维码。
- 如果需求没有提供二维码下方说明文字，不要自动生成。

## 宫格类型硬规则

宫格类型是第一硬条件，不允许跨宫格类型直接套用结构。

- 九宫格任务只能使用九宫格布局规则。
- 六宫格任务只能使用六宫格布局规则。
- 四宫格任务只能使用四宫格布局规则。
- 三宫格任务只能使用三宫格布局规则。
- 双图组合任务只能使用双图组合布局规则。

## 最终提示词规则

最终提示词必须把参考素材转写成外部 AI 平台可理解的完整结构描述。

最终提示词必须优先写结构硬约束，再写视觉风格。顺序建议为：

1. 先说明生成一张完整多宫格海报。
2. 再说明严格保持参考素材的宫格结构和模块位置。
3. 再逐格说明每格功能和内容。
4. 最后说明允许替换的视觉层内容。

不得写：

- 参考 G:
- 按本地路径图片生成
- 照搬参考图视觉风格
- 复刻参考图视觉风格

应写成：

```text
以多宫格参考素材为版式骨架，严格保持宫格结构、模块位置、产品卡排列、二维码位置、信息层级和裁切关系，只根据李宁品牌需求替换配色、字体、产品内容、人物素材、背景素材、节日元素、品牌视觉风格和场景氛围。
```

## 文案唯一来源规则

最终画面中出现的所有中文文案、数字信息、价格信息、权益信息、折扣信息、券包信息、促销标签、产品名、货号，必须来自运营需求表或用户明确提供的内容。

不允许 Codex 自行新增、改写、扩写、猜测任何文案。

如果需求没有提供价格、权益、折扣、券包等信息，最终提示词中不得出现任何价格类、优惠类、权益类文案。

## 输出要求

默认输出：

- 需求解析。
- 参考素材布局关系学习说明。
- 内部格子功能规划。
- 一张完整多宫格海报总提示词。
- 负向限制词。
- 简洁检查表。

不默认输出每格独立提示词，除非用户明确要求切图或每格单独生成。
