# Full Grid Prompt Rules

## 整张海报提示词必须说明

- 多宫格形式。
- 最终输出是一张完整海报图。
- 九宫格时，整体为 1:1 方形，内部为 3x3 网格布局。
- 整体视觉风格。
- 结构类型。
- 背景是否连续。
- 文字层级。
- 产品图使用方式。
- Logo 预留位置。
- 二维码是否需要出现。
- 如果需要二维码，二维码占用哪个完整格子。
- 不要拆成多张单独图片，除非用户明确要求。
- 最终外部生图提示词不得包含本地路径。
- 参考图结构必须转写为外部平台可理解的版式文字描述。
- 最终提示词必须表达为：保持参考图的信息布局和模块关系，但重新设计为李宁品牌视觉风格。
- 最终提示词必须明确严格保持布局结构、模块位置、产品区域、二维码区域和文字层级。
- 最终提示词必须明确不照搬原图配色、原图背景、原图人物、原图风格、原图光影和原图装饰元素。

## 最终生图提示词格式

整张海报输出必须分为两层：

A. 内部分析层：

- 可以记录本地参考图路径。
- 可以记录匹配到的模板编号。
- 可以记录参考图文件名。
- 可以记录参考图结构分析。

B. 外部生图提示词层：

- 不允许出现本地路径。
- 不允许写“参考某文件”“参考本地路径”“参考上图”“按照本地图片”。
- 必须把参考图内容转写成完整的文字描述。
- 必须可直接复制到其他 AI 生图平台使用。
- 必须说明“保持参考图的信息布局和模块关系，但重新设计为李宁品牌视觉风格”。

外部生图提示词必须包含画面比例与宫格形式、参考图结构转写后的版式描述、每格功能分布、主标题区位置、产品区位置、二维码区位置、文字层级、李宁品牌视觉风格、需求文案、产品要求和负向限制。

## 强锁定模式最终提示词表达规则

当参考图处于强锁定模式时，最终提示词必须明确说明：

- 这是基于参考图原有版式结构进行的视觉重设计。
- 严格保持参考图的整体构图、模块位置、产品位置、二维码位置、文字区域位置和信息层级结构。
- 只允许修改配色、字体设计、图片内元素、风格调性、人物素材和背景。
- 不允许重新设计版式。
- 不允许改变各模块之间的相对位置关系。
- 不允许调整产品卡的排列逻辑。
- 不允许改变二维码位置。
- 不允许改变主标题区和其他文字区的位置。
- 如果需求中没有提供价格 / 权益 / 券包文案，则只保留其结构位置，不生成任何文字。

## 文案强约束

- 整张海报中所有中文文案必须来自运营需求表或用户明确提供内容。
- 主标题、副标题、权益、价格、产品名、货号、行动引导必须逐字准确。
- 不允许自行新增、改写、扩写中文文案。
- 未提供的文案不得擅自补充，只能预留文字区域并标注“文案待运营确认”。
- 如果用户只提供“文案方向”，必须标注为方向，不得直接扩写成入画文案。
- 提示词必须写明禁止错字、乱码、漏字、多字和多余中文。

## 需求原文使用规则

- 最终提示词只能使用需求表或用户明确提供的文案。
- 不允许添加需求中没有的中文文案。
- 不允许添加需求中没有的价格、权益、券包、折扣、促销标签。
- 不允许添加需求中没有的产品名、货号。
- 不允许添加需求中没有的人物设定。
- 不允许添加需求中没有的场景设定。
- 如果需求没有提供某项内容，最终提示词中不能擅自补写。
- 如果必须保留参考图中的某个结构区域，但需求没有提供文字内容，只能写：“保留该结构位置，但不生成任何文字内容。”
- 不允许写“可添加促销文字”“可补充价格信息”“可加入优惠标签”“自动补充产品卖点”“自动生成活动口号”。

## 需求文案唯一来源规则

- 最终画面中出现的所有中文文案、数字信息、价格信息、权益信息、折扣信息、券包信息、促销标签、产品名、货号，必须来自运营需求表或用户明确提供的内容。
- 不允许 Codex 自行新增、改写、扩写、猜测任何文案。
- 不允许生成需求表中没有的“预估到手价”“到手价”“券后价”“限时优惠”“立减”“满减”“折扣”“券包”“爆款”“好价”“福利”“促销标签”“价格数字”“货号”“产品名称”。
- 如果需求表没有提供价格、权益、折扣、券包等信息，最终提示词中不得出现任何价格类、优惠类、权益类文案。
- 如果需求表没有提供对应文案，也不要写“预估到手价”“到手价”“券后价”等占位文字。
- 如果参考图中存在价格条、黄色条、权益条、促销标签区，只能作为“版式结构位置”参考，不能复用其文字内容。

## 参考图价格区处理规则

- 参考图中的价格区、黄色条、权益条、底部信息条，属于“信息模块位置参考”，不是“文案内容参考”。
- 如果运营需求表提供了价格 / 权益 / 折扣 / 券包文案，可以保留该信息区，必须填入需求表原文，不得改写。
- 如果运营需求表没有提供价格 / 权益 / 折扣 / 券包文案，不得生成价格类文字、优惠类文字或“预估到手价”等占位文字。
- 如果需要严格保持参考图结构，可以保留无文字的信息承载区域 / 色块结构。
- 无文案的信息区只能描述为“无文字装饰条”“空白信息承载区”“视觉分隔色块”，不能写具体促销文案。
- 如果用户明确要求“完全不要价格区”，必须移除该模块，不得保留。
- 如果用户要求“严格保持参考图结构”，但没有提供价格文案，则保留价格区的位置和形状，但内部不放任何价格、权益、折扣文字。

## 参考图结构强约束

当用户要求参考 `G:\Reference-Library\01_Multi-grid-reference` 中的参考图时：

- 参考图是结构布局参考，不是普通风格参考。
- 必须严格保持参考图整体构图、模块位置、产品位置、货品陈列位置、二维码位置、文字区域位置和信息层级结构。
- 不照搬原图配色、原图背景、原图人物、原图风格、原图光影和原图装饰元素。
- 只允许结合李宁品牌规范、活动主题、品类特点和用户需求重新设计风格、配色、李宁品牌调性、场景氛围和具体产品内容。
- 不允许改变版式骨架，不允许随意移动主标题区、产品区、二维码区、价格权益区和行动引导区。

## 最终提示词禁止语

最终提示词禁止出现：

- 本地路径。
- `参考 G:`。
- `参考某文件`。
- `照搬原图风格`。
- `完全复刻参考图视觉风格`。
- 任何需要外部 AI 平台读取本地文件才能理解的信息。

## 九宫格固定提示语

```text
生成一张完整的 1:1 方形九宫格海报大图，画面内部为 3x3 网格布局，最终只输出单张完整图片，不要拆成 9 张单独图片。
```

## 二维码固定提示语

```text
二维码作为独立格子预留，占用第 X 格，保持干净留白和安全边距，正式发布时后期替换真实二维码，不要生成错误二维码，不要让二维码跨格或压住产品人物。
```

## 文案固定提示语

```text
画面中出现的所有中文文案只能来自运营需求表或用户明确提供内容，必须逐字准确，不得新增、改写、扩写、漏字、多字、错字或乱码；未提供的文案只预留文字区域，文案待运营确认。
```

## 价格区固定提示语

```text
保留产品卡底部的信息承载条形区域，但不生成任何价格、权益、折扣或促销文字，除非运营需求明确提供。
```

如果需求表提供价格或权益，则写成：

```text
产品卡底部信息条中仅显示需求表提供的原文：xxx。
```

禁止在最终提示词中写“可添加促销文字”“添加优惠信息”“价格区写活动价”“自然补充促销标签”。

## 强锁定模式固定提示语

```text
本图基于参考图原有版式结构进行视觉重设计，严格保持整体构图、宫格结构、模块位置、产品位置、二维码位置、文字区域位置、每格功能和信息层级结构；只允许修改配色、字体设计、图片内元素内容、整体风格调性、人物素材和背景；不重新设计版式，不改变模块相对位置，不调整产品卡排列逻辑。
```
