# Single Grid Planning Rules

## 定位

本文件不再默认要求“每格单独生成图片”。

在多宫格任务中，每格说明默认只是完整海报内部的布局规划，用于帮助 AI 理解九宫格、六宫格、四宫格、三宫格或双图组合的画面结构。

## 内部格子规划必须包含

- 格子编号。
- 位置。
- 格子功能。
- 画面内容。
- 承载文字。
- 产品信息。
- 是否与整体大画面衔接。
- 是否为二维码独占格。

## 单格提示词脱路径规则

- 单格规划可以在内部分析层记录参考图路径和文件名。
- 每格外部生图提示词不得出现本地路径。
- 每格外部生图提示词必须把对应格子的参考结构转写成文字，例如产品在上方、价格权益在下方、二维码独占整格、主标题居中等。
- 不允许写“参考此路径”“参考本地图片”“参考上图”“参考文件”。
- 每格外部生图提示词必须表达为：保持对应格的信息布局和模块关系，但重新设计为李宁品牌视觉风格。
- 每格外部生图提示词必须明确不照搬原图配色、背景、人物、风格、光影和装饰元素。
- 禁止出现 `参考 G:`、`照搬原图风格`、`完全复刻参考图视觉风格`。

## 单格文案强约束

- 每格承载文字必须严格来自运营需求表或用户明确提供内容。
- 不允许为了填满某一格而自行新增中文文案。
- 不允许把“文案方向”扩写成具体入画文案。
- 如果该格需要文字但用户未提供，写“预留文字区域，文案待运营确认”。
- 如果用户提供了主标题、副标题、权益、价格、产品名、货号或行动引导，必须逐字使用，不得改写。

## 单格需求原文使用规则

- 每格提示词只能使用需求表或用户明确提供的文案。
- 不允许添加需求中没有的中文文案。
- 不允许添加需求中没有的价格、权益、券包、折扣、促销标签。
- 不允许添加需求中没有的产品名、货号。
- 不允许添加需求中没有的人物设定。
- 不允许添加需求中没有的场景设定。
- 如果需求没有提供某项内容，单格提示词中不能擅自补写。
- 如果必须保留参考图中的某个文字结构区域，但需求没有提供文字内容，只能写：“保留该结构位置，但不生成任何文字内容。”

## 单格价格权益禁生成规则

- 单格内所有数字信息、价格信息、权益信息、折扣信息、券包信息、促销标签、产品名、货号，必须来自运营需求表或用户明确提供内容。
- 不得生成需求表中没有的“预估到手价”“到手价”“券后价”“限时优惠”“立减”“满减”“折扣”“券包”“爆款”“好价”“福利”“促销标签”“价格数字”“货号”“产品名称”。
- 如果参考图对应格有价格条、黄色条、权益条、底部信息条，只能作为信息模块位置参考。
- 如果运营需求没有提供价格 / 权益 / 折扣 / 券包文案，该区域只能写为“无文字装饰条”“空白信息承载区”或“视觉分隔色块”。
- 如果用户明确要求“完全不要价格区”，该格必须移除该模块。

## 单格参考结构强约束

- 如果任务指定参考图，每格规划必须对齐参考图中对应格子的功能、模块位置、产品位置、文字区域和信息层级。
- 只允许替换具体产品，并结合李宁品牌规范、活动主题、品类特点和用户需求重新设计风格、配色和品牌氛围，不允许改变参考图版式骨架。
- 不允许照搬参考图原配色、原背景、原人物、原风格、原光影和原装饰元素。
- 如果参考图对应格是二维码格，该格仍必须作为二维码独占格预留。

## 单格强锁定模式

当参考图处于强锁定模式时，每格规划必须严格保持对应格的固定结构层：

- 格子位置不变。
- 格子功能不变。
- 模块位置不变。
- 产品位置不变。
- 文字区域位置不变。
- 价格 / 权益区位置不变；如果需求中没有文案，只保留位置，不生成文字。
- 二维码格位置不变。
- 信息层级不变。

每格只允许修改可变视觉层：

- 配色。
- 字体设计。
- 图片内元素内容。
- 风格调性。
- 人物素材。
- 背景。

每格提示词不得误导 AI 重新设计该格版式，不得改变该格与相邻格之间的拼接或模块关系。

## 单格独立提示词触发条件

只有用户明确提出以下要求时，才输出每格独立提示词：

- 每格单独生成。
- 导出 9 张、6 张、4 张、3 张或 2 张。
- 切图发布。
- 每格分别给不同平台使用。

如果用户没有明确要求，默认只输出整张完整海报提示词和内部格子规划。
