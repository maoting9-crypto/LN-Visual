# Template Matching Rules

## 模板库定位

模板库记录的是“布局模板”，不是“视觉风格模板”。

参考图只用于提取信息布局和模块关系，不用于照搬原图风格、配色、背景、人物、光影、品牌气质或装饰元素。

## 强锁定模式模板规则

如果用户启用强锁定模式，则模板必须完全继承参考图的结构层。

必须继承：

- 布局结构。
- 模块关系。
- 各区块位置。
- 产品摆放逻辑。
- 二维码位置。
- 信息层级。
- 拼接关系。

不得继承为固定项：

- 原图配色。
- 原图字体设计。
- 原图人物素材。
- 原图背景。
- 原图风格调性。

模板匹配成功后，输出时必须明确区分：

- 固定结构层。
- 可变视觉层。

强锁定模式下，风格不继承，结构必须继承；不能因为视觉重设计而改变参考图的版式结构。

## 用户指定参考图优先规则

用户指定参考图优先级最高。

指定方式包括：

- 完整路径。
- 文件名。
- 文件名关键词。
- 用户上传截图说明。

如果指定参考图存在，必须使用该参考图做结构分析。

如果指定参考图不存在，必须提示用户，不得自动换图。

如果未指定参考图，才进行自动模板匹配。

不能因为风格相似就替换掉用户指定参考图；版式结构优先于视觉风格。

## 模板提取内容

每次从参考图提取模板时，必须记录：

- 模板编号。
- 宫格形式。
- 整体布局结构。
- 每格功能。
- 主标题区位置。
- 产品区位置。
- 二维码区位置。
- 价格 / 权益区位置。
- 人物 / 场景区位置。
- 信息层级关系。
- 适合的运营需求类型。

## 模板禁止内容

模板不记录、不复用、不要求照搬：

- 原图配色。
- 原图背景。
- 原图人物。
- 原图风格。
- 原图装饰元素。
- 原图光影。
- 原图品牌气质。

## 匹配优先级
1. 宫格形式。
2. 信息布局。
3. 模块位置。
4. 产品位置。
5. 二维码位置。
6. 每格功能关系。
7. 文字层级结构。

不要因为风格相似就选择参考图。版式结构优先于视觉风格。

## 自动匹配输出要求

自动匹配时必须输出：

- 候选参考图列表。
- 匹配理由。
- 最终选择理由。

如果候选参考图超过 1 张且匹配不确定，必须先让用户确认，不允许擅自决定。

## 参考图价格区匹配规则

- 参考图中的价格区、黄色条、权益条、底部信息条，只能作为信息模块位置参考，不是文案内容参考。
- 模板匹配时可以记录这些模块的位置、形状、层级和与产品图的关系。
- 模板匹配不得把参考图中的“预估到手价”“到手价”“券后价”“折扣”“券包”“价格数字”“产品名”等内容迁移到新任务中。
- 如果运营需求表未提供价格 / 权益 / 折扣 / 券包文案，模板输出只能保留无文字的信息承载区域 / 色块结构。
- 如果用户明确要求“完全不要价格区”，模板匹配后也必须移除该模块。
- 如果用户要求“严格保持参考图结构”，但未提供价格文案，则只保留价格区的位置和形状，不放任何价格、权益、折扣文字。

## 新模板沉淀规则
如果参考图与现有布局模板相似，直接匹配旧模板；如果没有类似布局，则拆解并生成新模板编号。

新模板只沉淀信息布局和模块关系，不沉淀原图视觉风格。
