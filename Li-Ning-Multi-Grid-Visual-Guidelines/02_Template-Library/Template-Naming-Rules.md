# Template Naming Rules

## 模板库定位

模板库记录的是“布局模板”，不是“视觉风格模板”。

模板编号用于描述参考图的信息布局、模块关系和每格功能，不用于描述原图风格、配色、背景、人物、光影或装饰元素。

## 命名规则
模板编号由 Codex 自动命名，但必须可读、可检索、可复用。

格式：

```text
MG-宫格类型-结构类型-序号
```

示例：

```text
MG-9G-CENTER-THEME-001
MG-9G-PRODUCT-MATRIX-001
MG-6G-BIG-SCENE-001
MG-4G-PRODUCT-DISPLAY-001
```

## 模板必须记录

每个模板必须记录：

- 模板编号。
- 宫格形式。
- 整体布局结构。
- 每格功能。
- 主标题区位置。
- 产品区位置。
- 二维码区位置。
- 价格 / 权益区位置。
- 人物 / 场景区位置。
- 信息层级关系。
- 适合的运营需求类型。

## 模板禁止记录

模板不记录、不复用、不要求照搬：

- 原图配色。
- 原图背景。
- 原图人物。
- 原图风格。
- 原图装饰元素。
- 原图光影。
- 原图品牌气质。

## 命名建议

结构类型应优先使用布局含义，例如：

- `CENTER-THEME`：中心主题布局。
- `PRODUCT-MATRIX`：产品矩阵布局。
- `QR-PROMO-MATRIX`：二维码引流促销矩阵布局。
- `BIG-SCENE`：整体拼接大场景布局。
- `VERTICAL-SOCIAL-PROMO`：竖版社群主推 + 产品货盘布局。

不要用原图颜色、风格、人物或装饰元素命名模板。
