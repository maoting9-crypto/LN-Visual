# Reference Layout Analysis

## 使用的参考图

- 18点-朋友圈-示意图 (175).jpg

## 选择理由

- 用户明确指定该参考图为准。
- 该图为 2250x2250 的 1:1 方形九宫格完整大图，内部是清晰 3x3 网格。
- 结构与本次需求匹配：产品矩阵、人物 / 场景主视觉、中心主题区、二维码独占格同时存在。
- 该图可作为信息布局与模块关系参考，但最终视觉需重塑为李宁 618 奥莱与夏季跑步 / 速干品类风格。

## 参考图结构分析

这是 1:1 方形九宫格完整大图，画面内部为 3x3 网格。整体采用“产品卡片矩阵 + 连续人物场景 + 中心主题权益 + 右中二维码独占格”的结构。

## 每格功能划分

- 第 1 格（左上）：产品卡片区。上方为产品名称，中心为大比例产品图，底部为价格 / 权益条。
- 第 2 格（上中）：人物 / 场景主视觉区。与第 3、5、6 格共同形成连续场景背景。
- 第 3 格（右上）：人物 / 场景主视觉区。展示人物状态与环境氛围。
- 第 4 格（左中）：产品卡片区。结构与第 1 格一致，保持统一产品卡片系统。
- 第 5 格（中心）：主标题区 + 核心权益区。叠加在场景背景上，是整张图的视觉核心。
- 第 6 格（右中）：二维码区。二维码占据该完整格子的主要视觉面积，背景延续人物 / 场景主视觉，但二维码区域保持干净、安全、可替换。
- 第 7 格（左下）：产品卡片区。上方产品名，中间产品图，底部价格 / 权益条。
- 第 8 格（下中）：产品卡片区。上方产品名，中间产品图，底部价格 / 权益条。
- 第 9 格（右下）：产品卡片区。上方产品名，中间产品图，底部价格 / 权益条。

## 主标题区位置

- 主标题位于第 5 格中心，叠加在连续人物 / 场景背景上。
- 主标题为最大文字层级，承担活动主题识别。
- 核心权益 / 促销信息位于第 5 格下方，作为第二层级信息。

## 产品区位置

- 产品区位于第 1、4、7、8、9 格。
- 产品区采用统一卡片系统：上方文字、中间产品、底部条形信息区。
- 产品卡之间保持稳定边距、统一背景、统一产品比例。

## 人物区位置

- 人物区位于第 2、3 格，并延伸到第 5、6 格背景。
- 人物 / 场景区域形成 2 列 x 2 行的连续主视觉背景，但第 5 格叠加主题，第 6 格叠加二维码。

## 二维码区位置

- 二维码位于第 6 格右中完整格。
- 二维码必须保持独立完整格，不跨格，不压住产品和人物主体。
- 正式发布时后期替换真实二维码，不直接依赖 AI 生成可用二维码。

## 权益 / 价格区位置

- 第 5 格底部为核心权益 / 促销信息条。
- 产品卡片底部有价格 / 权益条，但本次需求未提供正式价格，因此最终提示词只保留条形信息容器，不生成未提供价格。

## 文字层级结构

- 第一层级：第 5 格主标题。
- 第二层级：第 5 格副标题或核心活动信息。
- 第三层级：产品卡片中的产品名称和 SKU。
- 第四层级：产品卡片底部价格 / 权益条或预留信息区。
- 二维码格内不新增扫码引导文案，除非需求表提供。

## 背景拼接关系

- 第 2、3、5、6 格共同组成连续人物 / 运动场景背景。
- 第 1、4、7、8、9 格为统一产品卡片背景系统。
- 九宫格之间用清晰分割线建立朋友圈九宫格结构。

## 整体构图骨架

- 左列上中两格是竖向产品陈列。
- 右侧上两格与中间两格形成连续运动场景和转化区。
- 中心第 5 格为主题核心。
- 右中第 6 格为二维码转化入口。
- 底排三格形成横向产品矩阵收口。
